This folder contains a **dataset suitable for this website**: eco-friendly, upcycled jewelry e-commerce. It is designed to improve the chat assistant (FAQs, intents, and optional model training).

## What's in the dataset

- **Domain:** Upcycled/recycled jewelry, sustainability, materials, care, shipping, returns, contact.
- **Intents covered:** `greeting`, `shipping`, `returns`, `materials`, `care`, `product`, `contact`, `payment`, `order_status` (matches your chat API).
- **Format:** `faq-dataset.json` â€“ one object per Q&A with `intent`, `question`, `answer`.
- **Size:** 2000+ pairs (generated by `scripts/generate-faq-dataset.js`). To regenerate or change size, run from project root: `node scripts/generate-faq-dataset.js`.

## How to use it

### 1. Seed your Supabase FAQs (no model training)

Use the dataset to add or expand FAQs so the existing **intent + FAQ** logic in `app/api/ai/chat/route.ts` gives better answers.

**Option A â€“ Manual:** Copy `question` / `answer` from `faq-dataset.json` into the Supabase `faqs` table (columns: `question`, `answer`, `sort_order`).

**Option B â€“ SQL:** From the project root, generate and run SQL:

```bash
# Node one-liner to output INSERT statements (run from project root)
node -e "
const d=require('./data/ai-training/faq-dataset.json');
let i=0;
console.log(d.pairs.map(p=>\"INSERT INTO faqs (question, answer, sort_order) VALUES ('\"+p.question.replace(/'/g,\"''\")+\"', '\"+p.answer.replace(/'/g,\"''\")+\"', \"+(++i)+\");\").join('\\n'));
"
```

Paste the output into the Supabase SQL editor and run (or use a seed migration). Skip any rows that duplicate existing FAQs.

### 2. Use with Ollama (free local AI)

The chat uses this dataset with a local model (Ollama). See **OLLAMA.md** in the project root: install Ollama, pull a model, set `.env`, and the app will use `faq-dataset.json` for FAQ-style answers. Product answers come from the live catalog only.

### 3. RAG (retrieval-augmented generation)

If you add RAG later (e.g. vector search over docs):

- Use each `question` + `answer` as a single chunk, or split long answers into short paragraphs.
- Embed chunks and store in your vector DB (e.g. Supabase pgvector).
- At chat time, retrieve relevant chunks and pass them into the existing LLM fallback in `app/api/ai/chat/route.ts`.

The same dataset is suitable as the main RAG content for support/FAQ-style answers.

## Adding more data

- Add more objects to `pairs` in `faq-dataset.json` with the same shape: `intent`, `question`, `answer`.
- Pull real questions from inquiries, reviews, or support chats (with permission) and add them as new pairs.
- Keep intents aligned with `route.ts`: `greeting`, `shipping`, `returns`, `materials`, `care`, `product`, `contact`, `payment`, `order_status`, `custom`, `unknown`.
