# AI Training Dataset - Plus Arch Upcycle

This folder contains the chatbot dataset and the files needed to prepare it properly for retrieval and fine-tuning.

## Files

- `faq-dataset.json`: raw generated Q&A pairs
- `faq-dataset.cleaned.json`: cleaned runtime dataset generated by `npm run ai:prepare`
- `fine-tune-train.jsonl`: model-ready training split generated by `npm run ai:prepare`
- `fine-tune-valid.jsonl`: validation split generated by `npm run ai:prepare`
- `training-report.json`: summary of dropped rows and intent counts

## Why preparation is required

The raw dataset is useful as a source file, but it contains synthetic paraphrases and occasional encoding issues. Training directly on it is likely to teach the model low-quality phrasing and stale product behavior.

The preparation step:

- repairs mojibake and punctuation issues
- removes obvious synthetic noise
- deduplicates questions
- exports JSONL in chat format
- excludes `product` intent from fine-tuning so live catalog answers stay grounded in Supabase

## Prepare the dataset

Run from the project root:

```bash
npm run ai:prepare
```

This generates the cleaned dataset and fine-tuning files in this folder.

## Runtime behavior

The chat API now prefers `faq-dataset.cleaned.json` automatically when it exists. If the cleaned file is missing, it falls back to `faq-dataset.json`.

## Fine-tuning guidance

Use the generated JSONL files as the starting point for any proper training pipeline.

- Train on `fine-tune-train.jsonl`
- Validate on `fine-tune-valid.jsonl`
- Review `training-report.json` before uploading the data

Keep live product data, prices, stock, and order status outside the fine-tune dataset. Those should stay grounded in the database or support workflow.

## Supabase FAQ seeding

If you want better FAQ retrieval without fine-tuning, you can still seed the `faqs` table from the cleaned dataset instead of the raw one.
